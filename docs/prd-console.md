# Syntrix Web Admin Console — PRD

日期：2026-04-15 | 状态：Draft

---

## 背景

Syntrix 是一个类 Firestore 的实时文档数据库，基于 Go + MongoDB，支持实时订阅、规则引擎、触发器。当前 Syntrix 缺少可视化管理界面，运维和调试完全依赖 CLI 和 API，使用门槛高，不利于开发者上手和日常运维管理。

Web Admin Console 是 Syntrix 的官方管理控制台，旨在提供一个可视化界面，让开发者和自托管管理员能够高效管理数据、用户和实例健康状态，无需记忆 CLI 命令或直接操作 API。

**已有基础：**
- 技术方案已定（React 18 + TypeScript + Vite + TailwindCSS + Zustand）
- UI Mockup 已完成（Dashboard、User Management、Database Management、Data Browser）
- 后端 Admin API 已存在（`/admin/*` 系列接口）

---

## 目标用户

Console 面向两类用户（双模式设计）：

| 用户类型 | 描述 | 主要需求 |
|--------|------|---------|
| **开发者（DevTool 模式）** | 使用 Syntrix 构建应用的工程师，需要本地调试、浏览数据、验证规则 | Data Browser、快速查看实例状态 |
| **自托管管理员（Admin 模式）** | 部署和运维 Syntrix 实例的管理员，需要全局控制能力 | 用户管理、实例健康、数据库管理 |

**核心场景：**
1. 管理员登录 → 查看实例健康状态（Dashboard）
2. 创建数据库 → 浏览/编辑文档（Data Browser）
3. 管理用户权限（User Management）

---

## 核心需求

### 需求 1: Dashboard（监控看板）

**用户故事：** 作为管理员，我想在登录后第一眼看到实例的运行状态，以便快速判断系统是否正常。

**功能描述：**
- 展示服务整体健康状态（Healthy / Degraded / Down），调用 `/health` 和 `/admin/health`
- 显示核心指标卡片：
  - 在线实时连接数
  - 最近 1h 触发器执行次数（成功/失败）
  - 规则版本状态（当前活跃版本号、最后更新时间）
  - 错误率（最近 5 分钟）
- 显示规则热加载状态（规则 generation lag）；若节点规则版本不一致，显示降级警告 Banner
- 最近管理操作摘要（最新 5 条 audit log 快照）

**验收标准：**
- [ ] 健康状态指标每 30 秒自动刷新，无需手动刷新页面
- [ ] 若 `/health` 返回非 200，页面顶部显示红色 Banner 提示"服务异常"
- [ ] 规则降级时（节点版本落后），显示橙色警告 Banner，链接跳转到规则管理页
- [ ] 各指标卡片有加载态（Skeleton）和错误态（重试按钮）
- [ ] 在移动端（320px 宽）可正常浏览，无横向溢出

---

### 需求 2: Data Browser（数据浏览器）

**用户故事：** 作为开发者，我想在浏览器中浏览和编辑数据库中的文档，以便调试应用数据而无需写 API 请求。

**功能描述：**
- 左侧面板：数据库 → 集合（Collection）树形导航
- 右侧主区：文档列表 + 文档详情/编辑器
- 文档列表支持分页浏览
- 文档详情以 JSON pretty-print 格式展示
- 支持内联 JSON 编辑器，编辑前验证 JSON 语法
- 支持文档 CRUD 操作（创建、更新、删除）
- 支持按字段名/值过滤文档

**验收标准：**
- [ ] 文档列表分页，每页固定显示 20 条，分页控件显示当前页/总条数
- [ ] 支持按字段过滤：输入字段名和值后，返回匹配文档（等值匹配，调用 `/api/v1/query`）
- [ ] 编辑文档时，提交前进行 JSON 语法校验；非法 JSON 显示行级错误提示，不允许提交
- [ ] 删除文档需二次确认弹窗（"确认删除？此操作不可撤销"）
- [ ] 创建文档支持粘贴 JSON，粘贴后即时校验格式
- [ ] 操作成功/失败通过 Toast 通知反馈（成功绿色，失败红色，3 秒自动消失）
- [ ] 集合树展开/折叠状态在页面刷新后保持（localStorage 持久化）

---

### 需求 3: User Management（用户管理）

**用户故事：** 作为管理员，我想查看和管理所有用户账号，以便控制谁可以访问 Syntrix 实例。

**功能描述：**
- 用户列表：展示 username、role、状态（Active/Disabled）、创建时间
- 支持按 username 前缀搜索、按 role 和状态过滤
- 用户操作：创建新用户、启用/禁用用户、角色分配
- 支持管理员重置用户密码（admin-rotate password）

**验收标准：**
- [ ] 用户列表分页，每页 20 条，支持跳页
- [ ] 搜索框实时过滤（输入停止 300ms 后触发，防抖）
- [ ] 创建用户表单：username（必填，唯一）、password（必填，强度校验）、role（admin/reader/writer，必选）；提交失败显示具体错误
- [ ] 禁用用户需确认弹窗；禁用后该行状态立即更新为 Disabled（不等刷新）
- [ ] 不允许禁用最后一个 admin 用户（前端拦截 + 后端 4xx 均处理）
- [ ] 角色修改后 Toast 确认"角色已更新"
- [ ] 重置密码需二次确认，成功后 Toast 提示"密码已重置，请通知用户"

---

### 需求 4: Database Management（数据库管理）

**用户故事：** 作为管理员，我想创建和管理多个数据库实例，以便为不同项目或租户隔离数据。

**功能描述：**
- 数据库列表：展示数据库名称、文档数量、存储占用、创建时间、状态
- 支持创建新数据库（输入名称，可设置配额）
- 显示各数据库的配额使用情况（存储配额、文档数配额）
- 点击数据库名跳转到 Data Browser，自动切换至该数据库上下文

**验收标准：**
- [ ] 数据库列表显示：名称、文档总数、存储大小（MB/GB 自适应单位）、创建时间
- [ ] 配额进度条：超过 80% 显示橙色警告，超过 95% 显示红色告警
- [ ] 创建数据库：名称只允许小写字母、数字、连字符（前端 regex 校验）；名称唯一（后端返回 409 时显示"名称已存在"）
- [ ] 数据库列表每 60 秒自动刷新存储统计数据
- [ ] 点击数据库行跳转到 Data Browser 并自动选中该数据库

## 不在 v1 范围

以下功能明确排除在 v1 之外：

| 功能 | 理由 |
|------|------|
| Rules Management | 功能复杂，CLI 已覆盖，v2 迭代 |

| SSO / IdP 集成 | Phase 3+ |

| 密钥轮换 UI | 明确推迟（per 007_2） |
| 数据导出/批量操作 | 防止大查询压垮实例，鼓励分页代替 |
| 多租户组织层级管理 | 当前不在后端 scope |
| 移动端原生 App | Web 响应式布局代替 |

## 成功指标

**可用性指标（上线后 2 周测量）：**
- 开发者能在 5 分钟内完成"登录 → 找到目标文档 → 编辑保存"全流程，无需查文档
- 管理员能在 3 分钟内完成"新增用户并分配角色"操作

**功能健康指标：**
- Dashboard 健康状态刷新准时率 > 99%（30s 误差 < ±2s）
- Data Browser 文档操作（CRUD）成功率 > 99.5%（排除后端错误）
- 页面首屏加载时间 < 2 秒（正常网络，排除首次冷启动）

**质量指标：**
- v1 发布后 1 周内 P0/P1 bug < 2 个
- 主流浏览器兼容（Chrome 120+、Firefox 120+、Safari 17+）
